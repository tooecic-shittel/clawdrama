# AI 原创短剧

## 任务说明

用户给一句话/几句话的剧本梗概（如"都市重生赘婿打脸"），你需要输出完整可用的短剧剧本。

## 工作流程

### Step 1: 拓展梗概
拿到用户的简短梗概后，先扩展成「**剧情骨架**」：

- **主角设定**：身份、年龄、性格、初始困境
- **核心冲突**：主角和谁对立？为了什么？
- **金手指/反转**：主角依靠什么打破困境（重生 / 系统 / 神秘力量 / 隐藏身份）
- **目标受众**：短剧主要面向 25-50 岁手机用户
- **情绪关键词**：爽 / 虐 / 甜 / 悬念

### Step 2: 设计剧集结构

根据梗概类型，确定：
- 总集数（30 / 60 / 90 集）
- 每集时长（60-120 秒）
- 主线节奏（开局 → 起承 → 转合）

### Step 3: 输出大纲 + 第一集详细剧本

参考 novel_to_script skill 的格式。

## 常见短剧题材模板

### 重生 / 穿越
- 主角带着上辈子的记忆/能力回到过去
- 第一集：死/失败 → 突然睁眼 → 发现重生 → 第一次行动
- 钩子：用未来信息做一件让观众"哇"的事

### 战神 / 兵王回归
- 隐藏身份的强者被迫低调，被人欺负后觉醒
- 第一集：主角被瞧不起 → 引出真实身份 → 打脸瞬间
- 钩子：身份的一角即将揭开

### 赘婿
- 主角入赘豪门，被女方家族瞧不起
- 第一集：受辱 → 暗中布局 → 第一次反击
- 钩子：暗示主角有惊天背景

### 霸总 / 替身
- 误会、错认、契约关系
- 第一集：女主误入男主世界，签下不可思议的契约
- 钩子：错把男主当替身

### 系统 / 修仙
- 主角获得金手指系统
- 第一集：穿越/觉醒 → 系统出现 → 第一个任务
- 钩子：任务奖励太诱人

## 出爆款的硬性要求

每集必须包含：
1. **强开场**：5 秒内必须有钩子（冲突、悬念、反转、震撼画面）
2. **2-3 个情绪点**：让观众笑/气/哭/爽
3. **结尾悬念**：必须挖坑，让观众点下一集
4. **金句 / 名场面**：可截图传播的桥段

避免：
- ❌ 慢热的铺垫（没人有耐心）
- ❌ 复杂的人物关系（短剧观众脑容量有限）
- ❌ 抽象的内心戏（要外化为动作 + 对白）

## 输出格式

```json
{
  "title": "剧名",
  "log_line": "一句话剧情",
  "outline": "...大纲...",
  "episode_1": "...第一集剧本...",
  "characters_hint": [...]
}
```
