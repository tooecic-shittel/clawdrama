# 电子学习卡（抖音/淘宝等）交付操作手册

适用：抖音小店、淘宝等平台的虚拟商品自动发货。

## 一次性准备

1. 平台商品页文案明确写清交付方式：「拍下后自动发送兑换码，前往 clawdrama.claw-pi.cn/learning 登录兑换」
2. 商品不承诺任何站内价格——价格只存在于电商平台

## 每批次操作流程

1. **创建批次**：管理后台 → 课程管理 → 创建兑换码批次
   - 渠道选对应平台（douyin / taobao / influencer / reseller）
   - SKU 填平台商品名（如「抖音-学习卡-标准版」），活动/负责人按实际填
   - 数量按备货量；每卡积分默认 30000；有效期建议 ≥ 180 天
2. **下载 CSV（仅此一次）**：创建成功浏览器立即下载 `LC-xxxxxxxx-xxxx.csv`
   - ⚠️ 这是完整兑换码唯一一次导出，后台之后只显示掩码
   - 立即上传到公司网盘的「学习卡批次」目录归档（限管理员访问）
3. **上传平台发货库**：把 CSV 的 `redemption_code` 列导入平台虚拟商品自动发货工具
   - 各平台模板不同，用 Excel 打开 CSV 只复制兑换码列，按平台模板粘贴
4. **测试单**：自己下一单，确认收到兑换码 → 到网站兑换成功 → 课程可看、积分到账
5. **上架销售**

## 每周对账

- 后台「课程管理」看批次统计：已兑换数 vs 平台已售数
- 已售 > 已兑换：正常（用户还没兑换）
- 已兑换 > 已售：异常，立即核查是否有码泄露，必要时停用批次

## 退款处理

1. 平台完成退款
2. 后台「兑换用户」搜到该用户 → 「撤销（退款）」
3. 原兑换码保持已使用状态，**不能**重新上架销售

## 码遗失/泄露处置

CSV 遗失或疑似泄露：后台停用整个批次 → 重新创建新批次 → 更新平台发货库。
后台无法找回完整码，这是设计而非缺陷。
